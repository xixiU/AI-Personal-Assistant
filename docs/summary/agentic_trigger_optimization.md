# Agentic 模式触发逻辑优化

## 优化概述

优化了 `AnthropicProvider` 的 `_should_use_agentic_mode()` 方法，使其能够更智能地识别代码排查意图。

## 修改文件

- `/mnt/d/source/AI-Personal-Assistant/src/ai_assistant/providers/anthropic_provider.py`

## 核心改进

### 1. 新增显式指令支持（最高优先级）

用户可以通过以下指令强制进入 Agentic 模式：

- `/code` - 使用代码工具
- `/排查` - 代码排查模式
- `/查代码` - 查找代码
- `/search` - 代码搜索（英文）

**示例**：
```
用户：/code 排查 /count/list 接口
系统：→ 强制进入 Agentic 模式
```

### 2. 增强明确意图词检测

新增以下意图关键词识别：

- 结合代码、查代码、看代码、读代码、分析代码
- 代码在哪、哪里实现、定位代码、找代码
- 代码排查、查看代码、检查代码

**示例**：
```
用户：请结合代码，帮我排查一下 /count/list 这个接口
系统：→ 检测到"结合代码"，进入 Agentic 模式
```

### 3. 新增技术特征词检测

自动识别以下技术特征：

#### 模块名
- 匹配以 `-service` 结尾的模块名
- 正则表达式：`[a-zA-Z0-9][\w-]*-service`
- 示例：`ts-service`、`user-service`、`api-gateway-service`

#### 接口路径
- 匹配以 `/` 开头的 API 路径
- 正则表达式：`/[a-zA-Z0-9/_-]{2,}`
- 示例：`/count/list`、`/api/login`、`/v1/users`

#### 版本号
- 匹配版本号格式（支持中文环境）
- 正则表达式：`(?<![a-zA-Z0-9])[vV]?\d+\.\d+(?:\.\d+)?(?![a-zA-Z0-9])`
- 示例：`4.3.6`、`v1.2.3`、`V4.0`

**示例**：
```
用户：ts-service 的 /count/list 接口有问题
系统：→ 检测到模块名和接口路径，进入 Agentic 模式

用户：就是4.3.6的，在ts-service下的
系统：→ 检测到版本号和模块名，进入 Agentic 模式
```

### 4. 新增上下文延续逻辑

检测历史消息（最近3轮）中是否提到代码相关意图，如果有则延续 Agentic 模式。

**示例**：
```
用户（第1轮）：请结合代码排查接口
系统：好的，请告诉我具体是哪个模块

用户（第2轮）：就是4.3.6的，在ts-service下的
系统：→ 上下文延续 + 技术特征，进入 Agentic 模式
```

### 5. 保持向后兼容

原有的触发条件仍然生效：

- 图片消息检测（日志截图）
- 排查关键词检测（排查、报错、异常、错误、bug 等）

## 触发优先级

1. **显式指令**（最高优先级）
2. **明确意图词**
3. **技术特征词**
4. **上下文延续**
5. **图片消息**
6. **排查关键词**（原有逻辑）

## 日志增强

每次触发 Agentic 模式时，会记录详细的触发原因：

```

触发 Agentic 模式：上下文延续（前一轮提到代码相关内容）
触发 Agentic 模式：检测到图片消息
触发 Agentic 模式：排查关键词
```

## 实现细节

### 辅助方法

新增了 4 个辅助方法：

1. `_has_explicit_command(text)` - 检测显式指令
2. `_check_code_intent_keywords(text_lower)` - 检测意图关键词
3. `_check_tech_features(text)` - 检测技术特征词
4. `_should_continue_agentic(messages)` - 检测上下文延续

### 正则表达式优化

所有正则表达式都经过测试，确保在中文环境下正常工作：

- 模块名：确保以字母或数字开头，避免误匹配中文前缀
- 版本号：使用负向断言而非 `\b`，兼容中文标点符号
- 接口路径：简单高效，匹配常见 API 路径格式

## 注意事项

1. 所有触发条件的前提是 **Git 工具已启用**（`git_tools_enabled=True`）
2. 日志记录了详细的触发原因，便于调试和分析
3. 触发逻辑按优先级顺序执行，匹配到第一个条件即返回
4. 保持向后兼容，原有触发逻辑不受影响
